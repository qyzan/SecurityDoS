Dokumentasi Profil Pengujian

Panduan lengkap mengenai profil pengujian statis dan mode kustom di Red Team.

Mode Baru

Custom (Manual)

Gunakan profil ini jika Anda ingin menentukan sendiri angka Throughput dan Durasi pengujian secara manual melalui Dashboard.

  • Unit RPS/TPS: Pilih apakah angka input adalah total request (RPS) atau target transaksi sukses (TPS).
  • Kontrol Penuh: Anda menentukan durasi dan beban maksimal sesuai kebutuhan spesifik.

Cocok untuk: Pengujian cepat untuk endpoint tertentu dengan beban yang sudah diketahui.

Mode Normal

Baseline (Referensi Performa)

Profil statis untuk mengukur performa sistem dalam kondisi normal sebagai titik acuan (Benchmark).

Tahap Durasi Beban
Baseline-Low 60s 50 RPS
Baseline-Medium 60s 200 RPS
Baseline-High 60s 500 RPS

Kasus penggunaan: Mengumpulkan metrik pembanding (Baseline) sebelum melakukan tes stress.

Default Headers: X-Test-Mode: Baseline, Accept-Language: en-US...
Statik (Standard)

Stress (Beban Bertahap)

Profil statis untuk mencari batas maksimal sistem dengan meningkatkan beban secara agresif.

Tahap Durasi Beban
Warm-up 30s 500 RPS
Load 1m 2.000 RPS
Stress 1m 8.000 RPS
Over-stress 30s 15.000 RPS
Maximum 10s 25.000 RPS

Kasus penggunaan: find breaking point dan perencanaan kapasitas infrastruktur.

Default Headers: X-Test-Mode: Stress, X-Performance-Check: true
Statik (Burst)

Spike (Lonjakan Mendadak)

Profil statis untuk menguji ketahanan sistem terhadap lonjakan trafik instan (Burst Limit).

Tahap Durasi Beban
Baseline 30s 300 RPS
Pre-spike 15s 800 RPS
SPIKE 10s 25.000 RPS
Post-spike 30s 800 RPS
Recovery 1m 300 RPS
Default Headers: X-Test-Mode: Spike, X-Injection: false
Statik (Adaptive)

Ramp (Peningkatan Adaptif)

Profil cerdas yang akan terus naik secara otomatis sampai terdeteksi kegagalan.

  • Start: 1.000 RPS
  • Peningkatan: +2.000 RPS setiap tahap (60 detik per tahap)
  • Batas Maksimal: 30.000 RPS
  • Auto-Stop: Berhenti otomatis jika Error Rate > 5%

Kasus penggunaan: Mencari angka pasti batas Rate-Limit server secara otomatis.

Default Headers: X-Test-Mode: Ramp, X-Adaptive-Control: active

Catatan Mode RPS vs TPS

  • RPS: Menembakkan permintaan tanpa peduli status sukses (Cocok untuk Flood Test).
  • TPS: Memastikan jumlah transaksi sukses (2xx) tercapai. Engine akan otomatis menaikkan RPS jika ada error (Success-rate feedback loop).
