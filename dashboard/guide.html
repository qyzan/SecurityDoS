Security DoS Guide

Platform Pengujian Ketahanan Infrastruktur & Keamanan Cyber v1.0

⬡ Overview

Security DoS adalah tool Load Testing dan DoS Simulation profesional yang dirancang untuk membantu tim keamanan (Red Team) dan DevOps mengukur batas maksimal infrastruktur mereka. Tool ini tidak hanya mengirimkan traffic, tetapi juga menganalisis perilaku server secara real-time untuk menemukan titik kegagalan (Breaking Point).

📊 Memahami Metrik

RPS (Requests Per Second)

Total upaya request yang dikirim ke server. Ini mencakup Sukses, Error, dan Timeout.

TPS (Transactions Per Second)

Menghitung semua transaksi yang Sukses (HTTP 2xx: 200, 201, 204, dll). Digunakan untuk mengukur kapasitas pengolahan data riil server.

Latency (ms)

Waktu respon server dalam milidetik. Kami mengukur hingga p99 (99% pengguna) untuk akurasi audit.

Active Workers

Jumlah antrean internal (Goroutines) yang sedang menunggu jawaban server. Jika penuh (10k), server dipastikan macet.

🎯 Profil Pengujian

  • Baseline: Pengujian traffic normal untuk menentukan standar performa sistem saat sehat.
  • Stress Test: Menaikkan beban secara bertahap hingga server tumbang untuk mengetahui kapasitas maksimal.
  • Spike Test: Memberikan beban kejutan yang sangat besar dalam waktu singkat (mensimulasikan serangan DoS tiba-tiba).
  • Ramp (Adaptive): Tool akan menaikkan RPS secara otomatis berdasarkan kemampuan server merespons.
  • Custom: Kebebasan penuh untuk mengatur RPS, Durasi, dan Timeout secara manual.

Pro Tip Tambahkan scenario_*.yaml ke folder configs/ untuk profil kustom baru.

User-Agent: Jika diisi, prefix kustom akan digunakan. Jika Evasion AKTIF, suffix acak akan ditambahkan otomatis.

Note: Input Header di UI akan menimpa (override) header bawaan profil jika Key sama.

🔍 Fitur Smart Analysis

Tool ini menggunakan algoritma analisis otomatis untuk mendeteksi batas operasional teknis dari sistem target:

1. Breaking Point (Threshold: 5%)

Berdasarkan ambang batas kesalahan (Error Rate). Sistem dianggap mulai gagal secara struktural jika tingkat kesalahan melampaui 5% secara konsisten selama minimal 3 detik. Kondisi ini biasanya menandakan kehabisan sumber daya (CPU/RAM/Cores) pada server backend atau kelebihan batas koneksi database.

2. Latency Degradation (Threshold: Configurable, default 2.0s)

Pemantauan kinerja ketersediaan berbasis waktu respons. Latensi rata-rata di atas nilai Latency Threshold yang Anda atur di Dashboard (default: 2000 ms) diidentifikasi sebagai titik degradasi layanan yang signifikan. Keterlambatan ini sering kali memicu penumpukan antrean koneksi (Connection Pool Exhaustion) yang mengarah pada kelumpuhan sistem (Cascading Failure).

3. Security Interception (Threshold: 15%)

Deteksi pembatasan sistem keamanan. Jika lebih dari 15% dari total traffic dibalas dengan kode HTTP blokir (contoh: 429 Too Many Requests atau 403 Forbidden), sistem menyimpulkan adanya perangkat perlindungan aktif yang menahan laju serangan (seperti Rate Limiter, WAF, atau DDoS Protection).

⚡ Emergency Control (Kill Switch)

Tombol Kill Switch adalah fitur keamanan prioritas tinggi yang dirancang sebagai "Rem Darurat" untuk menghentikan pengujian secara instan.

  • Berhenti Instan: Berbeda dengan tombol "Stop" biasa, Kill Switch menghentikan semua goroutine pengirim traffic dalam waktu < 1 detik.
  • Status Terkunci (Locked): Setelah diaktifkan, dashboard akan masuk ke status Kill Switch Active dan memblokir upaya pengaktifan tes baru untuk mencegah kesalahan operasional.
  • Cara Reset: Untuk kembali ke status normal, Anda cukup menekan tombol ⚡ Kill Switch sekali lagi (yang saat itu berwarna merah) untuk membuka kunci sistem.

Gunakan fitur ini segera jika Anda melihat sistem target mulai mengalami kegagalan fatal yang tidak terduga.

⚠️ Keselamatan & Etika

Gunakan tool ini hanya pada infrastruktur yang Anda miliki atau telah mendapatkan izin tertulis (Authorized Testing). Selalu pantau metrik real-time dan jangan tinggalkan sistem saat pengujian stress sedang berlangsung.

📚 Metodologi & Referensi Utama

Analisis dan parameter pengujian dalam Security DoS dikembangkan berdasarkan riset dan standar industri berikut:

  • Google SRE Book: Service Level Objectives — Dasar penentuan Error Budget dan ketersediaan sistem.
  • ISO/IEC 25010: Quality Models — Standar internasional untuk evaluasi kualitas sistem dan perangkat lunak.
  • Nielsen Norman Group: Response Time Limits — Standar UX untuk batas waktu tunggu manusia.
  • Google Web Vitals: LCP Thresholds — Standar modern untuk mengukur performa pemuatan halaman web.
  • Kubernetes Health Check Patterns — Metodologi Consecutive Failure untuk validasi status sistem.
